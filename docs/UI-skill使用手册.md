# UI Skill 使用手册

这份手册用于当前多智能体职业规划决策平台项目，帮助你在“非商业化、非主流、低 AI 感、可落地”的目标下，稳定地选择和使用合适的 skills 来设计与优化界面。

## 项目适配前提

- 项目类型：多智能体职业规划决策平台演示项目
- 技术栈：Vue 3 + TypeScript + Vite + Element Plus + Pinia
- 数据来源：本地 mock
- 目标气质：非商业化网页、非主流网页、减少常见 AI 生成界面的模板感
- 参考方向：多智能体职业规划决策平台，但不做传统企业后台的视觉复制

## 推荐优先级

### 第一优先级

#### `ui`

适合：
- 做页面视觉方向探索
- 想降低页面的 AI 感
- 想让课程系统出现更强的风格识别度
- 想尝试不那么像标准后台的布局与组件表达

特点：
- 覆盖页面、组件、布局、配色、图表、卡片、表单、导航等常见 UI 任务
- 更容易产出有辨识度的方案，不容易直接滑向“普通 SaaS 后台”

建议用途：
- 首页工作台风格探索
- 学生端学习中心卡片重构
- 管理端统计页视觉层级重做
- 课程卡片、信息面板、导航结构的风格化设计

#### `frontend-design`

适合：
- 把风格方案变成网页级落地结构
- 产出更接近真实前端界面的布局与组件组织
- 在“有风格”和“可用性”之间做平衡

特点：
- 强调高质量前端界面实现
- 适合作为 `ui` 之后的第二步，把抽象方向变成能开发的界面方案

建议用途：
- 把某个页面重做成完整可实现方案
- 设计新的仪表盘布局、课程详情页、管理台页面
- 做复杂模块的组件层级划分

#### `web-design-guidelines`

适合：
- 对已有页面做 UI/UX 审查
- 排查哪些地方 AI 感重、信息层级混乱、视觉节奏失衡
- 提升真实产品感和可用性

特点：
- 更偏审查与优化，不是单纯产出新视觉
- 很适合在页面初稿完成后使用

建议用途：
- 审查仪表盘页面的信息密度是否合理
- 检查按钮、卡片、图表、筛选区是否视觉主次清晰
- 优化移动端和桌面端断点体验

### 第二优先级

#### `theme-factory`

适合：
- 做多主题方向探索
- 快速建立一套带气质的色彩和字体策略
- 给课程系统做“人格化视觉外观”

建议用途：
- 学术档案风
- 校园策展风
- 实验室控制台风
- 编辑部信息版面风

注意：
- 它擅长做主题，不是专门负责创造组件结构

#### `vue-best-practices`

适合：
- 当你开始真正修改 Vue 页面时，保证实现方式稳健
- 在做较大 UI 改造时保持 Composition API、类型与组件结构清晰

建议用途：
- 页面重构
- 可复用组件拆分
- 复杂筛选面板、统计面板、角色工作台等实现约束

### 第三优先级

#### `canvas-design`

适合：
- 先做视觉语言草图
- 探索更强风格的版式、情绪与视觉构成

适合的情境：
- 你还不确定页面方向，想先做概念视觉
- 想探索“课程专题页”或“首页视觉封面”气质

局限：
- 不如 `frontend-design` 贴近真实前端页面结构

#### `web-artifacts-builder`

适合：
- 当你想快速做一个独立原型来试风格
- 需要更复杂的交互原型时

局限：
- 更偏原型与复杂 artifact，不是你当前 Vue 项目的首选常规工作流

## 最推荐的组合使用顺序

### 组合 A：做新页面风格

1. 先用 `ui` 生成 2 到 3 套方向
2. 再用 `frontend-design` 选定 1 套做成网页级方案
3. 再用 `web-design-guidelines` 审查并修正 AI 感和可用性问题
4. 进入项目开发时，再结合 `vue-best-practices`

### 组合 B：优化现有页面

1. 先用 `web-design-guidelines` 对当前页面做审查
2. 再用 `ui` 重新设计局部模块
3. 最后用 `frontend-design` 收敛成完整页面方案

### 组合 C：先找主题人格，再做页面

1. 先用 `theme-factory` 定义页面气质
2. 再用 `ui` 设计主要模块
3. 再用 `frontend-design` 落成页面结构

## 适合本项目的风格方向

### 学习档案馆风

特点：
- 像课程成长记录册
- 强调编号、标签、进度、卡片归档感
- 页面像档案索引，而不是企业后台

适合页面：
- 学生端首页
- 学习报告页
- 课程详情页

### 校园策展风

特点：
- 更像展览导视与专题编排
- 强调拼贴式版面、分区标题、非对称布局
- 视觉上更有“校园项目展”气质

适合页面：
- 首页
- 专题课程页
- 活动展示或作品展示区

### 实验室控制台风

特点：
- 保留管理系统效率感
- 但减少传统后台的重卡片和商务图表气质
- 偏克制、理性、略带研究装置感

适合页面：
- 管理后台统计页
- 教师端课程管理页
- 系统状态总览页

### 编辑部信息版面风

特点：
- 强调排版、信息等级、编号系统、索引感
- 能明显降低“AI 最爱的大圆角大阴影大渐变”的模板感

适合页面：
- 仪表盘
- 通知页
- 课程概览页

## 降低 AI 感的具体原则

- 避免全站统一大圆角卡片堆叠
- 避免每个模块都用相同阴影和相同留白节奏
- 避免过度依赖渐变背景和悬浮玻璃感
- 避免首页出现标准 SaaS 式三栏统计卡 + 图表 + 待办模板
- 提高信息结构的真实度，让每块内容的层级和密度更像真实系统
- 适当引入编号、索引、标签、批注、状态条等“信息设计元素”
- 通过不同区域的布局节奏变化建立页面个性，而不是只换颜色
- 图表区和操作区要有明确主次，不能所有模块都一样抢眼

## 直接可用的 Prompt 模板

### 给 `ui` 的提示词

```text
请为一个 Vue 3 + TypeScript + Element Plus 的多智能体职业规划决策平台设计 3 套低 AI 感的 UI 方向。

要求：
1. 不是商业化 SaaS 官网风，也不是普通企业后台风。
2. 页面气质偏非主流、非商业化网页，但仍要可用。
3. 参考多智能体职业规划决策平台的信息结构，包括课程、学习进度、教师反馈、系统统计、角色工作台。
4. 请重点降低以下常见 AI 感：统一大圆角卡片、过度渐变、模板化统计卡、千篇一律的 dashboard 布局。
5. 优先考虑以下风格方向：学习档案馆风、校园策展风、实验室控制台风、编辑部版面风。
6. 请输出每套方案的：配色思路、版式结构、组件风格、适合页面、风险点。
```

### 给 `frontend-design` 的提示词

```text
请把“实验室控制台风”的多智能体职业规划决策平台工作台页面，整理成一个可落地的前端页面方案。

项目背景：
- Vue 3 + TypeScript + Vite + Element Plus
- 中文界面
- 三角色：student、teacher、admin
- 当前为本地 mock 数据，不接真实后端

要求：
1. 页面是系统内页，不是营销首页。
2. 保留多智能体职业规划决策平台的效率感，但避免标准企业后台模板。
3. 需要包含：顶部摘要、核心统计区、课程/任务区、近期动态区、快捷操作区。
4. 请说明每个区域的布局关系、信息主次、推荐组件表现方式。
5. 请强调如何在桌面端和移动端都保持可用。
6. 请避免生成高度同质化的 AI 风 dashboard。
```

### 给 `web-design-guidelines` 的提示词

```text
请从 UI/UX 角度审查一个多智能体职业规划决策平台页面，重点检查以下问题：

1. 是否存在明显 AI 感，如模板化卡片堆叠、样式过于平均、视觉层级虚假丰富。
2. 信息密度是否真实，是否像真实产品，而不是为了“好看”而过度装饰。
3. 页面中各个模块的主次关系是否明确。
4. 图表、表格、操作按钮、筛选区之间是否存在视觉竞争。
5. 桌面端与移动端的结构是否合理。
6. 如何在不大改技术栈的前提下，让页面更有辨识度。

请按“问题 - 原因 - 优化建议”的形式输出。
```

### 给 `theme-factory` 的提示词

```text
请为一个多智能体职业规划决策平台设计 3 套不同的视觉主题，每套主题都要降低 AI 感，并适合中文信息型网页。

要求：
- 不要做商业化 SaaS 配色
- 不要做千篇一律的蓝紫渐变
- 每套主题都要说明：主色、副色、强调色、背景、边框、文字层级、适合的组件气质
- 主题方向优先考虑：学术档案、校园策展、实验室控制台、编辑部排版
```

### 给 `vue-best-practices` 的提示词

```text
请从 Vue 3 Composition API + TypeScript 最佳实践角度，审查一个多智能体职业规划决策平台页面重构方案。

重点检查：
1. 组件拆分是否合理
2. 状态是否应该留在页面内，还是进入 Pinia store
3. 是否符合 setup store / script setup 的风格
4. 是否容易因为 UI 重构导致类型、路由、角色权限或 mock 数据脱节
5. 如何在不引入新状态管理方案的前提下保持结构清晰
```

## 实战建议

- 想先找感觉：优先用 `ui`
- 想把方案落成页面：再用 `frontend-design`
- 想判断当前页面哪里最像 AI 生成：用 `web-design-guidelines`
- 想进入真实 Vue 改造：补上 `vue-best-practices`
- 想做一套更统一的人格化外观：加上 `theme-factory`

## 最终建议

如果你现在只想先挑一条最有效路径，建议直接使用这套顺序：

1. `ui`
2. `frontend-design`
3. `web-design-guidelines`
4. `vue-best-practices`

这套组合最适合你当前这个多智能体职业规划决策平台项目，也最容易做出“有辨识度但不失控”的页面结果。
