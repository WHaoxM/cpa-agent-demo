# 学生端功能详细总结

---

## 一、功能概览

学生端共包含 **12 个功能页面**，围绕"**职业探索 → 能力评估 → 技能提升 → 成果沉淀**"四阶段设计。首页通过智能引导卡和四步流水线将各功能串联为完整的职业规划闭环。

---

## 二、各功能详细说明

### 1. 首页（指挥中心）

- **路由**：`/app/dashboard`
- **功能描述**：
  - 作为整个学生端的中枢入口，负责展示当前用户的职业规划进度和学习概况
  - **智能引导卡**：根据用户完成情况自动判断当前处于四步流水线的哪一步，动态展示引导文案和跳转按钮。判断依据为：
    - 第 1 步"探索市场"：检查是否已关注职业方向（学习状态管理器的目标方向列表是否有数据）
    - 第 2 步"了解自己"：检查是否已解析简历或生成过报告
    - 第 3 步"提升技能"：检查是否有学习记录
    - 第 4 步"沉淀成果"：检查是否已生成报告
    - 全部完成后显示"继续学习之旅"，引导回到技能提升页
  - **四步进度条**：可视化展示"探索市场 → 了解自己 → 提升技能 → 沉淀成果"的完成状态，每步显示成果摘要（如"已关注 3 个职业方向"）
  - **学习概览统计**：已完成课程数、连续学习天数、平均分
  - **功能卡片入口**：职业分析、技能自评、职途导航、技能提升、我的报告、AI 助手共 6 张卡片，点击跳转对应页面
  - **视觉特色**：深舱空间背景（多层纵深、体积光、浮尘粒子）、层叠中控卡片（玻璃质感、光边框、错层悬浮）、滚动渐显动效

---

### 2. 职业分析（羊皮卷舆图）

- **路由**：`/app/student/career-analysis`
- **功能描述**：
  - 帮助学生**探索和了解各职业方向的市场行情**，是四步流水线的第 1 步
  - **5 大职业方向**：前端开发、后端开发、测试开发、数据分析、机器学习工程师
  - **15 个细分岗位**：每个方向下有 3 个具体岗位（如前端方向下有 React 前端工程师、Vue 前端工程师、可视化工程师）
  - **交互式气泡图**：用 D3 力导向布局绘制行业气泡图，按领域分色，点击气泡选中具体岗位
  - **中国地图热力图**：基于 ECharts 渲染 34 个省份的岗位需求热度，选中岗位后可点击省份查看该省详情
  - **三阶段操作流程**：
    1. 从气泡图选择岗位
    2. 从地图选择省份
    3. 查看深度洞察数据
  - **数据面板展示**：
    - 方向概览（全国平均中位薪资、在线岗位总数、年增长率）
    - 省份需求排行榜（前 10 名）
    - 省份薪资对标箱形图（全国最高 vs 全国均值 vs 当前选中省）
    - 薪资分档柱状图（初级/中级/高级）
    - AI 智能评价（整体趋势、供需分析、薪资建议、发展前景）
  - **关注职业方向**：页面右上角提供"关注"按钮，将当前方向写入学习状态管理器的目标方向列表。这是数据流的关键写入点，后续心仪岗位页和职业报告页都会读取此数据
  - **出口跳转**：
    - → 职途导航（"立刻上路"按钮）
    - → 职业能力图谱（"进入能力中心"按钮，携带当前方向参数）
  - **视觉特色**：宣纸纹理背景、水墨散点气泡图、卷轴式地图、楷体字标注（此视觉风格仅限本页面）

### 3. 技能自评

- **路由**：`/app/exams`
- **功能描述**：
  - 让学生**量化自己在目标岗位相关技能上的掌握程度**
  - **三阶段流程**：
    1. **选择方向**：从 5 大职业方向中选择，每个方向下展示细分子方向（如后端方向下有 Java 后端、Go 后端、C++ 开发、Python 后端等）
    2. **逐题自评**：针对所选子方向的核心技能逐一评分。每题提供 5 个能力等级：
       - 没听说过（0 分）
       - 了解概念（25 分）
       - 能做简单任务（50 分）
       - 独立完成项目（75 分）
       - 精通并能指导他人（100 分）
    3. **查看结果**：展示各技能得分
  - **题库规模**：每个子方向包含 10-15 道技能评估题，涵盖该方向的核心技术栈
  - **出口跳转**：
    - → 职业分析（"查看市场行情"按钮，携带当前方向标签）
    - → 职途导航（"开始求职规划"按钮）
  - **⚠ 注意**：评估结果不写入任何持久化状态管理器，页面关闭后数据丢失。也没有其他页面主动跳转到此页面（仅从首页和侧栏进入）

---

### 4. 职途导航（简历导入与解析）

- **路由**：`/app/student/career-navigation`
- **功能描述**：
  - 作为四步流水线的第 2 步"了解自己"，**通过简历解析帮助学生获得能力评估的数据基础**
  - **简历输入方式**：
    - 文件上传（支持拖拽）：接受 Word 文档、PDF、PPT、纯文本格式
    - 文本粘贴：直接粘贴简历内容
  - **模拟解析流程**：上传后自动执行三阶段解析动画（空闲 → 解析中 → 完成），模拟 AI 提取简历中的技能标签、项目经验、教育背景等信息
  - **职业方向选择**：解析完成后可选择一个目标职业方向，系统据此生成匹配分析
  - **解析结果写入**：所有解析结果持久化到简历状态管理器，包括原始文本、文件名、技能标签列表、职业匹配洞察
  - **出口跳转**：
    - → 个人能力画像（解析完成后自动跳转）
    - → 快速制作简历（"没有简历？点击快速制作简历吧！"提示按钮）
  - **入口来源**：首页引导卡、职业分析页、心仪岗位页、个人能力画像页返回、职业报告页返回、快速简历制作页返回

---

### 5. 个人能力画像

- **路由**：`/app/student/career-portrait`
- **功能描述**：
  - **前置条件**：必须先在职途导航页完成简历解析（简历状态管理器的已解析标志为真），否则自动重定向回职途导航页
  - **多智能体渐进式分析**：模拟 4 个 AI 智能体依次分析简历，每个阶段完成后立即渲染对应区域：
    1. **解析阶段**：提取个人基本信息（姓名、学校、专业、GPA）、项目经历、荣誉证书
    2. **评估阶段**：量化多维能力（如编程能力、沟通能力、学习能力等），生成能力维度列表
    3. **分析阶段**：计算综合竞争力指数、识别技能标签及权重
    4. **总结阶段**：生成 AI 综合评价（支持流式打字机效果输出）
  - **D3 雷达图**：可视化展示各能力维度的得分分布
  - **报告保存**：分析完成后可一键保存画像到报告状态管理器（类型为"画像"），我的报告页可查看
  - **出口跳转**：
    - → 职业生涯发展报告（"查看职业发展路径"按钮）
    - → 快速制作简历（"优化我的简历"按钮）
    - → 职途导航（"返回"按钮）

---

### 6. 职业能力图谱（三种布局模式）

- **路由**：`/app/student/career-ability`（通过 `?role=xxx` 参数指定职业方向）
- **功能描述**：
  - **将某个职业方向所需的全部能力以图谱形式可视化展示**，让学生直观看到需要掌握哪些技能、技能之间的依赖关系
  - **三种布局模式**（同一路由内切换，不使用子路由）：
    - **单图模式**：全屏展示能力节点关系图
    - **双栏模式**：左侧图谱 + 右侧节点详情对照
    - **工作台模式**：完整分析工作区，包含图谱和扩展信息
  - **能力节点**：按分组着色（如基础能力、框架能力、工程能力等），显示节点名称、权重
  - **已掌握技能高亮**：对比简历状态管理器中的已解析技能列表，已掌握的节点做高亮标记
  - **轨道布局算法**：使用自研轨道布局将能力节点排列为环形分层结构
  - **出口跳转**：
    - → 课程体系图谱（"进入课程体系"按钮，携带当前方向参数）
  - **入口来源**：职业分析页（"进入能力中心"按钮）

---

### 7. 课程体系图谱

- **路由**：`/app/student/course-system`（通过 `?role=xxx` 参数指定职业方向）
- **功能描述**：
  - **以三维空间形式展示某个职业方向对应的课程学习路径**
  - **Three.js 三维渲染**：课程节点在三维空间中分层排列，支持鼠标旋转、缩放、拖拽视角
  - **课程分层**：基础层 → 进阶层 → 高级层，体现学习的递进关系
  - **课程分组**：必修课程、选修课程、实践课程等，按分组着色
  - **节点交互**：点击课程节点查看课程详情
  - **方向联动**：默认读取学习状态管理器的目标方向列表中的第一个方向，也可通过 URL 参数指定
  - **出口跳转**：
    - → 技能提升页（"前往学习"按钮，携带当前方向参数）
  - **入口来源**：
    - 能力图谱的双栏模式和工作台模式（"进入课程体系"按钮）
    - 技能提升页（"返回图谱"按钮，形成双向跳转）

---

### 8. 技能提升（学习中心）

- **路由**：`/app/student/learning`（通过 `?role=xxx` 参数筛选对应方向课程）
- **功能描述**：
  - 作为四步流水线的第 3 步"提升技能"，**提供课程浏览和学习入口**
  - **课程库浏览**：展示所有可学习的课程，支持分类筛选和关键词搜索，分页展示
  - **课程卡片**：每张卡片展示课程封面、名称、时长、评分、学生人数
  - **D3 树图**：以矩形树图可视化课程分类结构和课程数量占比
  - **D3 弦图**：展示课程与技能之间的关联关系
  - **方向联动**：当从课程体系图谱或职业报告跳转来时，自动按该方向筛选相关课程
  - **双向跳转**：
    - → 课程体系图谱（"返回图谱"按钮，携带当前方向参数）
    - ← 课程体系图谱（携带方向参数跳入）

---

### 9. 职业生涯发展报告

- **路由**：`/app/student/career-report`
- **功能描述**：
  - 作为四步流水线的第 4 步"沉淀成果"，**生成完整的职业发展规划报告**
  - **岗位画像库**：基于目标方向展示不同层级的岗位画像（实习 → 初级 → 中级 → 高级 → 负责人 → 专家），每个岗位包含七维能力要求、核心技能、薪资范围、岗位描述
  - **七维能力模型**：专业技能、证书资质、创新能力、学习能力、抗压能力、沟通能力、实习经验
  - **D3 雷达图对比**：将学生当前能力（从简历解析推导）与目标岗位要求叠加对比，直观展示差距
  - **D3 力导向图**：以力导向布局展示岗位之间的晋升和转岗路径网络
  - **职业路径**：定义晋升路线（如初级前端 → 中级前端 → 高级前端）和转岗路线（如前端 → 全栈），标注每步需要补充的技能
  - **成长计划**：为学生生成短期和中期的成长行动方案，包含目标、任务清单、里程碑
  - **报告保存**：一键保存到报告状态管理器（类型为"职业"），我的报告页可查看
  - **心仪岗位方向优先级**：读取简历状态管理器的评估方向字段，在气泡布局中给该方向的岗位加权
  - **出口跳转**：
    - → 心仪岗位（"查看心仪岗位"按钮）
    - → 我的报告（"我的报告"按钮）
    - → 技能提升（"去学习"按钮，携带当前选中岗位名称）
    - → 职途导航（"返回"按钮）

### 10. 我的报告

- **路由**：`/app/student/my-reports`
- **功能描述**：
  - **以书架形式展示所有已保存的报告**，分为"个人能力画像"和"职业生涯发展"两个书架区
  - **书架式 3D 展示**：每份报告以书本形态摆放在虚拟书架上
  - **翻书式查看**：点击书本后以翻书动效打开报告详情覆盖层，展示完整报告快照
  - **报告数量统计**：标题旁显示各类报告数量
  - **入口跳转**：
    - → 职途导航（"开始测评"按钮，用于生成新的能力画像）
    - → 职业报告（"开始测评"按钮，用于生成新的职业报告）

### 11. 心仪岗位

- **路由**：`/app/student/favorites`
- **功能描述**：
  - **展示学生在职业分析页面关注的职业方向**，并为每个方向提供市场数据参考
  - **数据来源**：读取学习状态管理器的目标方向列表（由职业分析页写入）
  - **市场数据卡片**：每个已关注方向展示薪资范围、中位薪资、需求等级、热门城市、热门行业、技能标签、示例岗位、趋势说明、参考匹配度
  - **排序功能**：支持按匹配度或中位薪资升序/降序排列
  - **角色专属配色**：不同职业方向使用不同主题色（如前端开发为暖橙色、后端开发为墨绿色）
  - **出口跳转**：
    - → 职途导航（"进行人岗匹配"按钮，同时写入简历状态管理器的评估方向字段）
    - → 职业分析（"查看市场分析"按钮）

### 12. 快速制作简历

- **路由**：`/app/student/resume-builder`
- **功能描述**：
  - **为没有简历的学生提供在线简历编辑工具**，采用三栏布局
  - **模块化表单**（左栏编辑）：
    - 基本信息：姓名、电话、邮箱、学校、专业、年级、目标岗位、GPA、个人简介
    - 技能标签：以标签形式添加技能
    - 工作经历：公司、职位、时间段、描述（可添加多条）
    - 项目经验：项目名称、担任角色、描述（可添加多条）
    - 教育背景：学校、专业、学位、时间段、GPA（可添加多条）
    - 荣誉证书：类型（证书/实习/奖项）+ 名称
    - 自我评价：集成 Tiptap 富文本编辑器
  - **实时预览**（右栏）：编辑内容实时渲染为简历样式
  - **使用简历**：编辑完成后将简历文本写入简历状态管理器的草稿字段，并跳转到职途导航页进行解析
  - **出口跳转**：
    - → 职途导航（"使用简历"按钮 或 "返回"按钮）
  - **入口来源**：职途导航页（"没有简历？"提示）、个人能力画像页（"优化我的简历"按钮）

### 13. AI 助手

- **路由**：`/app/student/ai-assistant`
- **功能描述**：
  - **对话式学习问答工具**，学生可随时提问关于职业方向或技能学习的问题
  - **流式输出模拟**：回答以打字机效果逐字显示
  - **思考过程展示**：回答前先展示"思考中"动画和思考步骤，可展开/折叠查看思考内容，显示思考耗时
  - **预设快捷提示**：提供多个快捷问题按钮，点击即发送
  - **消息持久化**：所有对话记录保存在学习状态管理器的 AI 消息列表中
  - **上下文感知**：读取目标方向列表和简历解析状态作为上下文，但不产生可被其他页面使用的输出

## 三、功能闭环关系

### 闭环一：四步职业规划主闭环

这是系统的核心闭环，由首页驱动，每步完成后数据自动反映到首页的进度判断中。

```
              ┌──────── 首页（检测进度、智能引导）────────┐
              │                                          │
              ↓                                          │
     ① 职业分析                                          │
     （关注目标方向 → 写入目标方向列表）                     │
              │                                          │
              ↓                                          │
     ② 职途导航                                          │
     （上传简历 → 写入简历解析结果）                        │
              │                                          │
              ↓                                          │
     ③ 技能提升                                          │
     （学习课程 → 产生学习记录）                            │
              │                                          │
              ↓                                          │
     ④ 职业报告                                          │
     （生成报告 → 写入报告快照）                            │
              │                                          │
              └──── 全部完成 → 首页显示"继续学习" ──────────┘
```

**数据流说明**：

- 第 1 步完成标志：学习状态管理器的目标方向列表长度大于 0
- 第 2 步完成标志：简历状态管理器的已解析标志为真，或报告状态管理器的画像记录长度大于 0
- 第 3 步完成标志：学习状态管理器的学习历史长度大于 0
- 第 4 步完成标志：报告状态管理器的全部记录长度大于 0

---

### 闭环二：简历解析 → 能力评估 → 报告生成 子闭环

嵌套在主闭环的第 2 步和第 4 步之间：

```
     职途导航（简历解析）
       │            ↑ 返回
       ↓            │
     个人能力画像（多智能体分析 → 保存画像报告）
       │            ↑ 返回
       ↓            │
     职业报告（七维对比 → 保存职业报告）
```

**数据依赖说明**：

- 个人能力画像**强依赖**简历解析结果——未解析则自动重定向回职途导航
- 职业报告读取简历解析的技能数据来推导学生当前七维能力值
- 两者生成的报告都保存到报告状态管理器，在"我的报告"页统一展示

## 四、未形成闭环的独立功能

### 1. 技能自评

- **问题**：评估结果不写入任何状态管理器，页面关闭即丢失
- **问题**：没有其他业务页面会主动跳转到技能自评页
- **现状**：仅从首页和侧栏进入，完成后可跳到职业分析或职途导航，但自评数据不被任何下游页面消费
- **结论**：属于单向输出的断裂分支

### 2. AI 助手

- **现状**：独立的对话工具，虽然读取了目标方向和简历状态作为上下文，但不产生可被其他页面使用的结构化输出
- **结论**：纯辅助型工具页面，不参与业务闭环

### 3. 消息中心

- **路由**：`/app/messages`
- **现状**：页面内硬编码消息列表，不读写任何业务状态管理器
- **结论**：完全独立的通知展示页

### 4. 个人中心

- **路由**：`/app/profile`
- **现状**：头像上传和个人信息编辑，仅涉及用户基础信息
- **结论**：独立的账户设置页，与业务链路无关
