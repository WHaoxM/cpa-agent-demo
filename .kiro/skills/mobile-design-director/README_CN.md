# mobile-design-director

> **v5.0** — 描述需求或提供截图，获取公式化设计方案。

## 功能

移动端 UI/UX 设计总监 Skill。提供：

- **公式化设计方案** — 每个建议都是可见公式：`[1px内描边] + [噪点纹理] + [内发光] = 极致质感容器`
- **强制需求审查** — 做设计前必须整体思考需求
- **多方案输出** — 2-3 个方案，从不同维度切入
- **实现规格** — 精确数值、代码片段、拿来即用

## 使用方法

用自然语言描述即可：

```
"帮我设计一下这个页面"
"看看这个截图，怎么做更好看"
"重新做一下这个组件"
"设计整个功能的品牌体验"
```

或者提供截图要求 review/redesign。

## 工作流

```
Step 0: 需求全景 — 先整体 review 需求（必须）
Step 1: 诊断 — 分析现状
Step 2: 配方 — 构建 2-3 个公式化方案
Step 3: 裁决 — 专业推荐 + 实现规格
```

## 公式手册（示例）

| 公式 | 结果 |
|------|------|
| `[1px内描边]` + `[噪点纹理]` + `[内发光]` | 极致质感容器 |
| `[超大号背景排版]` + `[前景遮挡]` + `[Z轴分离]` | 杂志级破窗感 |
| `[情绪画布]` + `[物理样机]` + `[漂浮装饰物]` | Dribbble Shot |
| `[60-30-10分配]` + `[HSB状态]` + `[WCAG对比度]` | 色彩和谐 |
| `[弹簧动画]` + `[按压缩放]` + `[触觉反馈]` | 物理触感 |

完整手册包含 12+ 个公式，见 [SKILL.md](SKILL.md)。
